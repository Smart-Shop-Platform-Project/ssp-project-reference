Service catalog with UI connection types for Smart Shop Platform

Direct UI calls (sync REST — browser awaits response)
Auth Servicesync
Register, login, refresh tokens, logout. Issues JWT access and refresh tokens.
UI calls directly on login/register pages
Product Servicesync
Product catalogue CRUD, categories, pricing, image metadata. Reads from MongoDB with Redis cache.
UI calls on home, category, and product detail pages
Search Servicesync
Keyword and vector search over products using OpenSearch. Returns ranked product IDs.
UI calls when user types in the search bar
Cart Servicesync
Add/remove/update cart items in Redis. Verifies price with Product Service on each add.
UI calls on add-to-cart and cart page
Order Servicesync
Creates orders, returns order status. Writes to PostgreSQL, kicks off the order saga via Kafka.
UI calls on checkout and order history pages
AI Recommendersync
Returns personalised product recommendations using SageMaker, based on preference vectors built async.
UI calls for homepage/sidebar recommendation widget
Services the UI never calls directly (internal-only, async via Kafka)
Inventory Serviceasync
Consumes orders.placed, reserves/releases stock in PostgreSQL. No UI-facing endpoints.
Triggered only by Kafka events from Order Service
Payment Serviceasync
Consumes inventory.reserved, charges card via Stripe, publishes payment result. Lambda + Flask.
Triggered only by Kafka events, never by UI
Notification Serviceasync
Consumes payment.processed/failed, sends email/SMS via SES/SNS. Lambda + Flask.
Triggered only by Kafka events
Fraud Detection AIsync (internal)
Called synchronously by Order Service during checkout to score risk before accepting an order.
Called by Order Service, not by UI directly
