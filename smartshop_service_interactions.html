Smart Shop Platform service interaction low-level design — sync and async flows for every major user action

Interaction types:
syncHTTP REST call — caller waits for response
asyncKafka event — fire and forget
DBdirect database / cache read/write
retryfailure path with retry/rollback
Flow 1 — user login
POST /api/v1/auth/login → JWT returned in <200ms
1
Browser POSTs credentials to ALB → routed to API Gateway
2
sync API Gateway forwards to Auth Service (FastAPI) on port 8000 via internal ALB target group
3
DB Auth Service queries PostgreSQL (users table) — bcrypt hash comparison
4
DB On success: writes session token to Redis (TTL=24h) for token revocation support
5
Signs and returns JWT (access_token 15min + refresh_token 7d). No other service is called.
Flow 2 — browse and search products
GET /api/v1/products → product list
1
API Gateway validates JWT (local RS256 verify — no auth service call)
2
sync Routes to Product Service (FastAPI)
3
DB Checks Redis cache (key: products:page:1:limit:20). Cache hit → return in ~2ms
4
DB Cache miss → queries MongoDB (DocumentDB), writes result to Redis (TTL=5min)
5
sync Calls AI Semantic Search Lambda in parallel if query param present — kNN OpenSearch lookup
AI recommendation sidebar load
1
Browser makes separate GET /api/v1/recommendations?user_id=X
2
sync API Gateway → AI Recommender Lambda invoked synchronously (function URL or API GW)
3
DB Lambda reads user preference vector from Redis (pre-computed by async consumer)
4
sync Calls SageMaker endpoint with preference vector → returns ranked product IDs in ~80ms
5
Returns top 8 product IDs → client fetches details from Product Service
Flow 3 — add to cart
POST /api/v1/cart/items → cart updated (pure Redis, no DB)
1
sync API Gateway → Cart Service (Flask on ECS)
2
sync Cart Service calls Product Service to verify product still exists and get current price (prevents stale price in cart)
3
DB Cart stored as Redis Hash: key=cart:{user_id}, field=product_id, value=JSON({qty, price, added_at}). TTL=7 days.
4
Returns updated cart. No Kafka event — cart is ephemeral until checkout.
Flow 4 — place order (the saga — most complex flow)
POST /api/v1/orders → order confirmed (saga pattern across 5 services)
1
sync API Gateway → Order Service (FastAPI). Validates JWT, reads cart from Cart Service.
2
sync Order Service calls Fraud Detection Lambda synchronously. Returns risk_score. If score > 0.85 → reject order immediately.
3
DB Order Service writes order to PostgreSQL with status=PENDING. Returns order_id to client immediately (202 Accepted).
4
async Order Service publishes event to Kafka topic orders.placed with key=order_id.
5
async Inventory Service (consumer group: inventory-grp) picks up event → checks MongoDB stock → reserves items → writes to PostgreSQL inventory_reservations → publishes to inventory.reserved.
6
async Payment Service Lambda (consumer group: payment-grp) picks up inventory.reserved event → calls Stripe API → publishes to payment.processed or payment.failed.
7
async Order Service (consumer group: order-update-grp) reads payment.processed → updates order status to CONFIRMED in PostgreSQL.
8
async Notification Service Lambda reads payment.processed → sends order confirmation email via SES.
9
async AI Recommender Lambda reads orders.placed → updates user preference model in Redis for future recommendations.
!
retry If payment fails: Payment Service publishes to payment.failed. Order Service updates status=PAYMENT_FAILED. Inventory Service reads payment.failed → releases reservation. Notification sends failure email.
Flow 5 — admin updates a product
PUT /api/v1/products/{id} → product updated across all caches and search index
1
sync API Gateway → Product Service. Writes updated product to MongoDB.
2
DB Product Service invalidates Redis cache keys: products:*, product:{id}.
3
async Publishes event to products.updated Kafka topic.
4
async Search Service (consumer group: search-grp) reads event → re-indexes product document in OpenSearch including new embedding vector.
5
async Inventory Service reads event → updates local product price cache if needed.
