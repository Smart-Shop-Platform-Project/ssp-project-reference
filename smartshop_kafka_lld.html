Kafka low-level design for Smart Shop Platform — brokers, topics, partitions, producers, and consumers

MSK cluster — 3 brokers, 2 AZs (cost-optimised)
3 brokers is the minimum recommended for fault tolerance. Each partition has 1 leader and 2 replicas spread across brokers. If broker-1 goes down, a replica on broker-2 or broker-3 is immediately elected leader — zero message loss (replication.factor=3, min.insync.replicas=2).

Producer → topic → consumer group wiring
Producer Order Service
Publishes to orders.placed
Key = order_id → same partition for same order
acks=all · retries=3 · idempotent=true
Consumer group inventory-grp
Inventory Service reads orders.placed
3 consumers → 3 partitions (1:1 ideal)
Reserves stock, publishes to inventory.reserved
Consumer group recommender-grp
AI Recommender reads orders.placed
Separate group = independent offset tracking
Updates user preference model async
Consumer group payment-grp
Payment Service reads inventory.reserved
Only charges card after stock is confirmed
Saga pattern: reserve → pay → confirm
Consumer group notification-grp
Notification Service reads payment.processed
Sends order confirmation email/SMS
auto.offset.reset=earliest for retry safety
Consumer group search-grp
Search Service reads products.updated
Re-indexes product in OpenSearch on change
Keeps search index eventually consistent
Kafka producer config (aiokafka — order-service)
producer = AIOKafkaProducer(
    bootstrap_servers="broker1:9092,broker2:9092,broker3:9092",
    acks="all",              # wait for all ISR replicas to confirm
    enable_idempotence=True, # exactly-once semantics, no duplicates
    retries=5,
    retry_backoff_ms=300,
    compression_type="lz4",  # reduces payload size ~60%
    max_in_flight_requests_per_connection=1,  # preserve ordering
    value_serializer=lambda v: json.dumps(v).encode("utf-8"),
    key_serializer=lambda k: k.encode("utf-8"),
)
Kafka consumer config (aiokafka — inventory-service)
consumer = AIOKafkaConsumer(
    "orders.placed",
    bootstrap_servers="broker1:9092,broker2:9092,broker3:9092",
    group_id="inventory-grp",          # unique per service
    auto_offset_reset="earliest",      # never miss a message on restart
    enable_auto_commit=False,          # manual commit after processing
    max_poll_records=50,
    session_timeout_ms=30000,
    heartbeat_interval_ms=10000,
)

async def consume():
